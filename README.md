```
## PARTE 2: Configuração do GitHub Projects

### Opção 1: GitHub Projects (Recomendado para SDC)

**Setup Inicial:**

1. **Criar Project Board**
   - Ir em "Projects" > "New project"
   - Escolher template: "Board" (tipo Kanban)
   - Nome: "SDC 2026 - Sprint Board"

2. **Colunas Sugeridas:**
```
📋 Backlog → 🔜 To Do → 🏃 Doing → 👀 Review → ✅ Done → 📦 Archived
```

**Descrição das Colunas:**

- **Backlog:** Todas as ideias e tarefas futuras
- **To Do:** Tarefas priorizadas para o sprint atual
- **Doing:** Trabalho em andamento (limite: 2 cards por pessoa)
- **Review:** Aguardando revisão do time/supervisor
- **Done:** Concluído no sprint
- **Archived:** Histórico (mover ao final de cada sprint)

3. **Labels Essenciais:**
``` 
    
    🔴 urgente          - Deadline próximo
    🟡 bloqueado        - Impedimento
    🔵 pesquisa         - User research
    🟢 design           - Design/prototipagem
    🟣 desenvolvimento  - Código
    🟠 documentação     - Papers, docs
    ⚪ reunião          - Meetings, alinhamentos
    🏷️ ethics           - Questões éticas
    📝 submission       - Materiais de entrega
